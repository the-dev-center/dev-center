= VCS Provider Profiles

DevCentr stores provider-specific configuration in `app/src/modules/vcs/profiles.json5`. This file defines how the app interacts with different Git hosting services (GitHub, GitLab, Gitea, etc.), including organization profile repositories and README formats.

== Organization Profile Support

Some providers allow organizations to display a custom README on their profile page. The format (repo names, README paths, URL patterns) differs by provider. DevCentr reads this from each provider's `orgProfile` section.

Only providers that support org-level profile READMEs have an `orgProfile` section. Bitbucket and generic Git do not.

=== GitHub

* **Public profile:** `org/.github` (must be a public repo), README at `profile/README.md`
* **Private profile:** `org/.github-private`, README at `profile/README.md`
* **Profile URLs:** `https://github.com/{org}?view_as=public`, `https://github.com/{org}?view_as=member`

=== GitLab

* **Single profile:** `gitlab-profile` repo, README at root `README.md`
* No separate public/private profile repos; all group members can view and edit

=== Gitea

* **Public profile:** `org/.profile`, README at root
* **Private profile:** `org/.profile-private`, README at root
* Base URL uses `{host}` for self-hosted instances (e.g. `codeberg.org`)

== profiles.json5 Structure

Each provider entry includes:

* `displayName` — Human-readable name
* `hostPattern` — Host to match (e.g. `github.com`, `gitlab.com`)
* `release` — Release tagging commands and view URLs
* `orgProfile` (optional) — Organization profile config:
** `publicRepo` — Repo name for public profile
** `privateRepo` — Repo name for member-only profile; `null` if not supported
** `readmePath` — Path to README within the repo (e.g. `profile/README.md`)
** `readmeFormat` — Usually `markdown`
** `baseUrl` — Base URL with `{org}` and `{host}` placeholders
** `publicViewParam` / `memberViewParam` — Query params for profile page views

== Adding a New Provider

To add org profile support for another provider (e.g. Codeberg, Forgejo):

1. Add an entry in `profiles.json5` with `hostPattern` and `orgProfile`.
2. Use the provider's documentation for the exact repo names and README locations.
3. The repository browser will automatically show the Organization Profile section when an owner is selected for that host.
