= Compilers on Windows: Clang vs. GCC
:navtitle: Compilers on Windows

When developing high-performance software on Windows, selecting the right compiler toolchain is critical for both runtime performance and developer experience.

== Why Clang is Preferred

For modern Windows development, **Clang** (part of the LLVM project) is generally preferred over GCC for several reasons:

* **Native Compatibility**: Clang is designed to be compatible with Microsoft's Visual C\+\+ (MSVC) ABI. This means binaries compiled with Clang can interoperate seamlessly with native Windows libraries and tools.
* **Superior Diagnostics**: Clang providing significantly clearer and more actionable error messages and warnings compared to traditional GCC versions.
* **High Performance**: Clang is built for high-performance software, often providing better optimization passes for modern CPU architectures.
* **Tooling Integration**: LLVM-based tools (like `clang-format`, `clang-tidy`, and LLD) provide a cohesive ecosystem that integrates well with modern IDEs and CI/CD pipelines.

== The GCC & MSYS2 Confusion

While GCC is the standard on Linux, using it on Windows often leads to a convoluted environment:

* **The MSYS2 Necessity**: To use GCC to compile native Windows executables (via MinGW-w64), you typically need **MSYS2**.
* **Complexity & Confusion**: MSYS2 provides multiple "environments" (UCRT64, MINGW64, CLANG64, MSYS). This often confuses developers who find themselves accidentally linking against the POSIX-compatibility layer (`msys-2.0.dll`) instead of creating truly native Windows binaries.
* **Path Pollution**: MSYS2 environments often rely on complex shell configurations and path management that can conflict with other Windows-native tools.

== Recommended Toolchain

We recommend using the official LLVM distribution for Windows. It provides a clean, native experience without the overhead of a POSIX-emulation environment.

[source,powershell]
----
# Install the full LLVM toolchain via winget
winget install LLVM.LLVM
----

This installation includes `clang`, `clang++`, and the `lld` linker, which are ready to use from any standard PowerShell prompt.
