= Writing Clear Docs with AI
:navtitle: Writing Clear Docs with AI
:description: Prompt hack for AI-generated documentation that stays understandable.

When you use AI to draft or rewrite documentation (vibe-coding, Copilot, Cursor, or any LLM), the result is often dense or assumes too much. One prompt hack consistently produces the clearest, most scannable articles:

**Ask the AI to write for "80 IQ people."**

That instruction (or similar: "write so someone with no background can follow," "explain every term the first time," "short sentences, one idea per step") pushes the model toward:

* Short sentences and one idea per paragraph or step.
* Explaining jargon the first time it appears (e.g. "DNS provider = the place where you manage your domain").
* Numbered steps in the right order, with no skipped assumptions.
* Tables and lists for "what to type where" so readers can copy-paste.
* A quick "What do I need?" or summary at the top so people can jump to their case.

The goal is not to insult anyone; it is to force the AI out of its default "expert-to-expert" style. Writing for the lowest common denominator of clarity improves the doc for everyone.

In this repo, the rule is stored in `.cursor/rules/documentation-ai-clarity.mdc` so Cursor applies it when working here. You can add the same prompt to your global AI rules (e.g. Cursor user rules or a shared `rules.txt`) for other projects.
